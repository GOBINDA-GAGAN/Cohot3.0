# 1.  What is semantic HTML and why is it important?
- "Semantic HTML means using HTML tags according to their meaning and purpose instead of using generic tags like div and span everywhere.

- For example, tags such as header, nav, main, section, article, and footer clearly describe the content they contain.

- Semantic HTML is important because it improves accessibility, helps search engines understand the page structure better, and makes the code easier for developers to read and maintain."

# 2.Flexbox vs Grid 
- Flexbox and Grid are both CSS layout systems, but they solve different layout problems.

- I use Flexbox when I need to arrange items in a single direction, either horizontally or vertically. It is best for components like navigation bars, buttons, cards, and form layouts.

- I use CSS Grid when I need to control both rows and columns at the same time. It is ideal for dashboards, galleries, and complete page layouts.

- A simple way to remember it is:

- Flexbox = One-dimensional layout (Row OR Column)

- Grid = Two-dimensional layout (Rows AND Columns)"
